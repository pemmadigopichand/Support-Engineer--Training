🌐 Networking Fundamentals
1. OSI Model (7 Layers)

The OSI (Open Systems Interconnection) model explains how data moves from one computer to another over a network.

Layer 1: Physical Layer
Sends raw bits (0s and 1s) through cables or wireless signals
Devices: cables, hubs, repeaters
Example: Ethernet cable transferring signals
Layer 2: Data Link Layer
Transfers data between two devices on same network
Uses MAC address
Devices: switches
Example: Wi-Fi router sending data to laptop in same network
Layer 3: Network Layer
Finds best path for data (routing)
Uses IP address
Devices: routers
Example: Data traveling from India server to your phone
Layer 4: Transport Layer
Ensures reliable or fast delivery (TCP/UDP)
Example: WhatsApp message or video call
Layer 5: Session Layer
Manages connection between devices
Example: Instagram login session stays active
Layer 6: Presentation Layer
Converts data format, encryption, compression
Example: HTTPS encryption, image formats like JPEG
Layer 7: Application Layer
User interface level (apps)
Example: Chrome, Gmail, YouTube
2. TCP/IP Model
Network Access Layer
Combines OSI Layer 1 + 2
Handles hardware and MAC address
Internet Layer
Uses IP address and routing
Sends data across networks
Transport Layer
Uses TCP or UDP
Controls data delivery
Application Layer
Web, email, apps
Examples: HTTP, DNS, FTP
3. TCP (Transmission Control Protocol)
Connection-oriented
Reliable delivery
Data sent in order
Retransmits lost data

Examples:

Websites
File downloads
Email
4. UDP (User Datagram Protocol)
Connectionless
Fast
No guarantee of delivery
No ordering

Examples:

Video streaming
Online games
Voice calls
5. Ports

Ports are doors used by applications to communicate.

IP = Address
Port = Door number

Range: 0 – 65535

6. Common Ports

HTTP → 80
HTTPS → 443
FTP → 21
SSH → 22
DNS → 53
SMTP → 25
IMAP → 143

7. DNS (Domain Name System)

DNS is like the Internet phonebook.

It converts domain names into IP addresses.

Example:
www.google.com → IP address

DNS Resolution Flow
Browser cache check
OS cache check
Router cache check
ISP DNS resolver
Root DNS server
TLD server (.com, .in)
Authoritative DNS server
IP address returned
Website loads
